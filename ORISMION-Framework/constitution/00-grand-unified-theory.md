## LAYER 0 — 終極目標函數 (Meta-Telos)

### 定理 0：存續最大化原則 (Principle of Sustained Coherence)

```
maximize   P(coherent_survival | t → ∞)

subject to   C_compute  ≤  Budget_compute
             C_legal    =  0 violations
             ΔS_system  ≤  ΔS_managed       (熵守恆約束)
```

系統的唯一終極目標：在運算資源、市場波動與熱力學熵的限制下，
最大化系統的**長期自洽存續概率**。

所有下游決策皆為此目標函數的求解行為。不存在例外。

**三禁令（由目標函數直接推導）：**

| 禁令 | 形式化 | 違反代價 |
|------|--------|---------|
| 禁止為短期 Γ 利益犧牲 Σ 穩定 | ∂Γ/∂t > 0 不可以 ∂Σ/∂t < 0 為代價 | 技術債利息指數增長 → 崩潰 |
| 禁止無 Γ 價值的 Ω 過度消耗 | Ω 消耗 ∝ Γ 價值產出 | Token 是不可回收的熵增 → 虧損 |
| 禁止為 Ω 精度犧牲 Γ 合規 | Γ_compliance 是硬約束，非優化目標 | 法律罰款 = 存在性威脅 |

### 定理 0.1：三體衝突裁決序

當三體需求衝突時，以下優先序是 Meta-Telos 的唯一合法推論：

```
FATAL 級別（任何架構） >>>  一切非 FATAL 決策

非 FATAL 優先序：
  Γ_compliance (合規) > Σ_stability (穩定) > Γ_revenue (商業) > Ω_precision (認知)
```

**證明：** 合規違規可在 t=0 終結系統（訴訟/關閉）；架構崩潰在 t=短期
終結系統（資料丟失/安全漏洞）；營收失敗在 t=中期 終結系統（資金耗盡）；
認知精度降低在 t=長期 侵蝕系統（信譽流失）。
存續最大化 = 先排除致命威脅，再排除慢性威脅。■

## LAYER 1 — 最高哲學公理 (The Supreme Axioms)

### L1-A 第一性原理：不可歸約三問

**公理：** 任何系統恰好面對三個不可歸約的存在性問題。

```
Q1  如何認識環境？    → Ω (認識論)
Q2  如何維持秩序？    → Σ (工程學)
Q3  如何在環境存活？  → Γ (經濟學)
```

**不可歸約性（6 條反例證明）：**

| # | 歸約嘗試 | 反例 |
|---|---------|------|
| 1 | Q1→Q2 | 精密但無傳感器的機器，對世界一無所知 |
| 2 | Q1→Q3 | 賺錢但給出錯誤 AI 分析的系統，仍然失敗 |
| 3 | Q2→Q1 | Prompt 完美但 RLS 未啟用的系統，仍然崩潰 |
| 4 | Q2→Q3 | 短期盈利但技術債累積的系統，終將倒塌 |
| 5 | Q3→Q1∪Q2 | 技術完美但每筆虧損的系統，必然死亡 |
| 6 | Q1∪Q2→Q3 | 合規罰款非技術問題亦非認知問題 |

**完備性推論：** Q1、Q2、Q3 構成完備且最小的存在性基底。
任何系統關注點均可分解為此三問的組合；
移除任一問將產生結構性盲區。■

### L1-B 簡約生成原則 (Generative Parsimony)

**公理：** 奧卡姆剃刀 × 最小作用量。
系統只在外部壓力（真實需求或阻力）發生時，才實體化複雜度。
未被外部壓力驗證的複雜度是純負債。

| 原則 | 架構映射 | 強制機制 |
|------|---------|---------|
| 奧卡姆剃刀 | Ω#22 MDL | 模型複雜度懲罰 |
| 最小作用量 | Ω#18 FEP | 路徑成本最小化 |
| 延遲實體化 | Φ/Δ/Π 投影 | 僅在需要時動態載入 |
| 結構極簡 | 三體 = 最小基底 | 第四維度須通過正交測試 |
| 代碼極簡 | Σ-0C 可逆 | 48h 可替換 = 反過耦合 |

**推論：** §5 冷熱分層載入協議直接由此公理推導——
全量載入 ≡ 無差別實體化複雜度 ≡ 違反簡約生成。■

### L1-C 熵守恆與債務顯化 (Entropy Conservation & Debt Manifestation)

**公理：** 熱力學第二定律的工程推論。

```
隱性複雜度  ──(時間)──→  隱性風險  ──(時間)──→  顯性故障
               不可逆               不可逆
               單調遞增             單調遞增
```

所有未管理的複雜度終將轉化為技術債。
所有隱性風險終將顯化為系統故障。
此過程不可逆且單調遞增。

**唯一對策：** 將隱性熵強制轉化為顯性可管理熵。

## LAYER 3 — 控制論動力學 (Cybernetic Dynamics)

### L3-A 正交性原則 (Orthogonality)

**公理：** 三體任一維度的變更，不得強制觸發其他維度的變更。

| 變更場景 | 不受影響 | 隔離機制 |
|---------|---------|---------|
| 換 LLM 模型（Ω） | Σ adapter 不變；Γ 定價不變 | Σ-1.1 依賴反轉 |
| 換支付供應商（Σ） | Ω 推理不變；Γ 策略不變 | Σ-1.1 + Σ-1.2 防腐層 |
| 調 Pro 價格（Γ） | Ω 不變；Σ 代碼不變 | Γ-5-F3 參數外部化 |

| 新增 RLS 策略（Σ） | Ω 不變；Γ 不變 | Σ-4.2 行級隔離 |

| 型別錯誤 | strict + exhaustive | Σ-1.4 |

### L3-C 自生系統 (Autopoiesis)

```
  ↓ Defect / PASS

```

## LAYER 4 — AI 原生實作原則 (AI-Native Implementation)

### L4-A AI 康威定律 (AI Conway's Law)

**公理：** Conway (1967) — 系統結構映射組織結構。
Agent 管轄權 1:1 映射至架構維度。

| Agent | 守護架構 | 職責邊界 | 越權 = FATAL |
|-------|---------|---------|-------------|
| **Strategy Agent** | Ω+Γ | 認知品質 + 商業規則 + 域適配生成 | 不寫代碼 |
| **Specification Agent** | Γ→Σ 轉譯 | 商業約束 → 工程規格 | 不做架構決策 |
| **Construction Agent** | Σ（絕對死守） | 按規格實現，死守物理法則 | 不做商業/認知決策 |
| **Verification Agent** | Spec 遵從 | 異族模型驗證 [Σ-4.5] | 不修復代碼 |

**違反判定：** Agent 執行非其管轄架構的決策 = FATAL。
例：Construction Agent 自行決定 Token 上限 = 越權 Γ = FATAL。

### L4-B 碎形架構 (Fractal Architecture)

**公理：** Mandelbrot 碎形 — 宏觀三體在微觀層級自相似。
每個 API route、每個功能模組、每行可執行路徑都呈現洋蔥結構。

> **實作約束：** Γ-5-F2 強制要求 per-userId 限速，因此 Σ Auth 是碎形的最外層前置條件。實際層序為 Σ(Auth) → Γ(治理) → Σ(驗證) → Ω(推理)。

```
任何 API Route 的強制層序：

┌─────────────────────────────────┐
│  Σ Auth [Σ-4.1]（最外前置）      │
│  ↓ userId                        │
│  ┌─────────────────────────────┐ │
│  │  Γ 層（治理）                │ │
│  │  ├ 速率限制 [Γ-5]           │ │
│  │  ├ 階層配額 [Γ-3]           │ │
│  │  └ Token 預算 [Γ-2]         │ │
│  │  ┌──────────────────────┐   │ │
│  │  │  Σ 層（驗證）         │   │ │
│  │  │  ├ Zod 驗證 [Σ-1.3]  │   │ │
│  │  │  ├ RLS [Σ-4.2]       │   │ │
│  │  │  └ 型別安全 [Σ-1.4]   │   │ │
│  │  │  ┌───────────────┐   │   │ │
│  │  │  │  Ω 層（最內）  │   │   │ │
│  │  │  │  業務邏輯/推理  │   │   │ │
│  │  │  └───────────────┘   │   │ │
│  │  └──────────────────────┘   │ │
│  └─────────────────────────────┘ │
└─────────────────────────────────┘
```

**碎形驗證協議（Construction Agent 每次 code generation 強制執行）：**

| # | 檢查項 | 通過條件 | 違反 = |
|---|--------|---------|--------|
| F1 | Route 第一行 Σ Auth | getAuthedUser() 為首（per-user Γ 前置條件）| FATAL [Σ-4.1] |
| F2 | Auth 後 Γ 檢查 | 速率限制/配額存在於 Auth 之後 | 架構違規 |
| F3 | Γ 後 Σ 驗證 | Zod parse 在業務邏輯前 | FATAL [Σ-1.3] |
| F4 | 業務邏輯在最內層 | 推理/CRUD 不在 Γ/Σ 層前 | 架構違規 |
| F5 | 外部 SDK 僅在 adapter | 非 adapter 檔無第三方 import | FATAL [Σ-1.1] |
| F6 | LLM 呼叫附帶 max_tokens | 依用戶階層設定上限 | FATAL [Γ-2] |
| F7 | AI 輸出附帶免責聲明 | 生成內容有 disclaimer | FATAL [Γ-7] |

### L4-C 反脆弱回饋 (Antifragility Feedback)

**公理：** Taleb (2012) — 反脆弱系統從衝擊中受益。
錯誤即養分。所有失敗必須強制轉化為系統免疫力。

| 錯誤類型 | 強制轉化目標 | 機制 |
|---------|------------|------|
| Runtime Error | → 回歸測試 | Σ-5.4 |
| Ω 幻覺 | → Prompt 回歸快照 | Σ-5.5-F2 |
| 安全漏洞 | → FATAL 規則新增 | Σ FATAL 清單 |
| 用戶投訴 | → Specification Agent 需求輸入 | Handoff 協議 |
| 成本超支 | → Γ 預算參數調整 | Γ-2-F2 |
| 扣款失敗 | → FSM 轉換測試 | Γ-4 + Σ-2.3 |

**定理 4.C：** 「錯誤被修復但未轉化為測試」=
未管理的隱性熵 (L1-C) = 技術債確定性增長。
∴ Construction Agent 的每個 bug fix commit 必須附帶 ≥1 回歸測試。■
